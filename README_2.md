# Лабораторна робота №2: Дослідження роботи з компонентом Fragment

## Мета роботи
Дослідити створення та взаємодію компонентів Fragment з Activity, набути практичних навичок у побудові динамічних інтерфейсів користувача в рамках однієї Activity.

## Виконання завдання (Варіант 14)

Згідно з варіантом №14, додаток реалізує інтерфейс для вибору книги (автора) та року видання за допомогою фрагментів.

### Реалізація вимог Task 2:
1.  **Одноекранна архітектура**: Робота з фрагментами відбувається в межах `MainActivity`.
2.  **InputFragment (Перший фрагмент)**:
    *   **Згорнутий список (Spinner)**: Містить перелік книг та їх авторів.
    *   **Група опцій (RadioGroup)**: Дозволяє обрати рік видання (радіо-кнопки).
    *   **Кнопка «ОК»**: Збирає дані з полів вводу та передає їх через інтерфейс до Activity.
3.  **OutputFragment (Другий фрагмент)**:
    *   **Текстове поле**: Відображає відформатований результат вибору користувача (Книга + Рік).
    *   **Кнопка «Cancel»**: Повертає користувача до першого фрагмента, видаляючи поточний, та ініціює очищення форми вводу.

## Ключові класи та методи

### `MainActivity.kt`
Головний контейнер для фрагментів, який координує взаємодію між ними.
*   `onDataPassed(text: String)`: Реалізація інтерфейсу для отримання даних з `InputFragment` та передачі їх в `OutputFragment` через `Bundle`.
*   `onCancelClicked()`: Реалізація інтерфейсу для обробки натискання кнопки "Cancel", яка видаляє `OutputFragment` зі стеку (`popBackStack`) та викликає метод очищення форми в `InputFragment`.

### `InputFragment.kt`
Клас, що відповідає за збір даних від користувача.
*   `setupSpinner()`: Налаштовує `ArrayAdapter` для випадаючого списку книг.
*   `handleOkButtonClick()`: Валідує вибір користувача (перевірка RadioGroup) та передає сформований рядок через `OnDataPassedListener`.
*   `clearForm()`: Публічний метод, який скидає стан Spinner та RadioGroup до початкового (використовується після натискання "Cancel").

### `OutputFragment.kt`
Клас для відображення результату.
*   `displayReceivedData()`: Отримує дані з аргументів (`Bundle`) та встановлює їх у `TextView`.
*   `setupListeners()`: Налаштовує обробник кнопки "Cancel", що сповіщає Activity про необхідність закриття фрагмента.

## Взаємодія між фрагментами
Взаємодія реалізована за класичною схемою: **Fragment -> Activity (через Interface) -> Fragment (через Bundle)**. Це забезпечує низьку зв'язність (loose coupling) компонентів додатка.

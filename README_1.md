# Лабораторна робота №1: Дослідження елементів керування Android

Цей проект є Android-застосунком, розробленим на мові **Kotlin**. Він демонструє базову взаємодію з елементами користувацького інтерфейсу (UI) та обробку подій.

## Реалізовані технології
- **Мова програмування:** Kotlin.
- **UI Framework:** Android XML Layouts (ConstraintLayout, LinearLayout).
- **Компоненти UI:** Spinner, RadioGroup, RadioButton, Button, TextView, ScrollView.
- **Діалогові вікна:** AlertDialog для валідації вводу.
- **Ресурси:** Використання `strings.xml` для локалізації та зручного керування текстом.

## Основні класи
- **`MainActivity`**: Головний клас активності, що успадковується від `AppCompatActivity`. Він керує життєвим циклом екрана та логікою взаємодії користувача з інтерфейсом.
- **`ArrayAdapter`**: Використовується для зв'язування масиву даних (списку книг) з елементом `Spinner`.
- **`AlertDialog.Builder`**: Служить для створення та відображення модальних вікон з попередженнями.

## Ключові методи
- **`onCreate(Bundle?)`**: Точка входу в активність. Тут виконується ініціалізація інтерфейсу через `setContentView`.
- **`findViewById(Int)`**: Метод для пошуку та отримання посилань на елементи інтерфейсу, визначені в XML.
- **`setupSpinner()`**: Налаштовує адаптер для випадаючого списку та заповнює його даними.
- **`setOnClickListener { ... }`**: Встановлює обробник натискання на кнопку "ОК".
- **`handleOkButtonClick()`**: Основний метод бізнес-логіки:
    - Перевіряє, чи обрано рік у `RadioGroup`.
    - Якщо дані відсутні, викликає `showWarningDialog`.
    - Якщо дані коректні, формує рядок результату та відображає його в `tvResult`.
- **`getString(Int, ...)`**: Отримує форматований рядок з ресурсів проекту, що дозволяє безпечно вставляти динамічні дані в текст.

## Валідація
Згідно з технічним завданням, застосунок перевіряє вибір користувача. Якщо жоден з варіантів у групі радіо-кнопок не обраний, користувач бачить спливаюче вікно з проханням завершити введення даних.

# Лабораторна робота №3: Дослідження способів збереження даних

Цей проект демонструє реалізацію системи керування персональною бібліотекою з використанням локального сховища даних. Додаток розширює функціонал попередньої лабораторної роботи, додаючи можливість постійного зберігання, оновлення та видалення записів.

## Виконання вимог Task 3

У поточному коді вимоги завдання №3 реалізовано наступним чином:

1.  **Збереження даних (Сховище)**:
    *   В якості сховища використано реляційну базу даних **SQLite**, управління якою здійснюється через клас `DatabaseManager`.
    *   Для початкового наповнення бази даних використано файл `assets/books.csv`, що дозволяє автоматично заповнити бібліотеку при першому запуску.

2.  **Запис результату та інформування**:
    *   **Кнопка «OK»**: Роль кнопки підтвердження виконує кнопка **"Save"** у діалоговому вікні додавання книги. При її натисканні дані з полів вводу (назва, автор, рік) передаються в БД через метод `insertBook`.
    *   **Повідомлення про успіх**: Після успішного запису до бази даних спрацьовує механізм зворотного зв'язку через інтерфейс `OnDataPassedListener`, який викликає `Toast` повідомлення у `MainActivity` з текстом "Added: [Назва книги]".

3.  **Перегляд даних та навігація (Кнопка «Відкрити»)**:
    *   **Кнопка «Відкрити» / Навігація**: Замість окремої діяльності (Activity), у додатку реалізовано сучасний підхід з використанням **ViewPager2** та **TabLayout**. Перехід на вкладку "Statistics" або повернення до "Library" виконує функцію переходу між різними представленнями даних сховища.
    *   **Вміст сховища**: Вкладка "Library" відображає список усіх книг у `RecyclerView`, а вкладка "Statistics" показує агреговані дані (загальна кількість, прочитані/непрочитані).
    *   **Порожнє сховище**: Якщо в базі даних немає записів, `RecyclerView` залишається порожнім, а статистика відображає нульові значення.

4.  **Додатковий функціонал**:
    *   **Оновлення**: Користувач може змінювати статус книги (прочитано/не прочитано) безпосередньо у списку.
    *   **Видалення**: Реалізовано функцію видалення книги з бази даних натисканням на відповідну іконку у списку.

## Ключові класи та методи

### Класи роботи з даними
*   **`DatabaseManager`** — центральний клас для роботи з SQLite.
    *   `onCreate()`: створює структуру таблиць.
    *   `seedDatabaseFromCSV()`: імпортує початкові дані з активів.
    *   `insertBook(book: Book)`: додає новий запис до таблиці `books`.
    *   `getAllBooks()`: повертає повний список книг.
    *   `updateBookStatus(id, isRead)`: оновлює стан прочитання книги.
    *   `deleteBook(id)`: видаляє запис за ідентифікатором.
*   **`Book`** — Data-клас, що описує модель книги (id, title, author, year, isRead).

### Класи інтерфейсу (UI)
*   **`InputFragment`** — відповідає за відображення списку книг та логіку додавання нових записів.
    *   `showAddBookDialog()`: створює та показує `AlertDialog` для введення даних.
    *   `refreshList()`: оновлює дані в адаптері з урахуванням вибраних фільтрів.
*   **`OutputFragment`** — відповідає за відображення аналітичних даних.
    *   `updateStatistics()`: зчитує актуальні дані з БД та оновлює текстові поля статистики.
*   **`BookAdapter`** — пов'язує список об'єктів `Book` з елементами `RecyclerView`.
    *   `onStatusChanged`: лямбда-функція для обробки зміни стану книги.
    *   `onDeleteClicked`: лямбда-функція для видалення запису.

### Навігація
*   **`MainActivity`** — хост-активність, що координує взаємодію між фрагментами та керує відображенням `Toast`.
*   **`LibraryPagerAdapter`** — забезпечує роботу `ViewPager2`, перемикаючи фрагменти списку та статистики.
